# htmlslide (Dynamic Slide System)

Markdown原稿とアセット群をブラウザにドラッグ＆ドロップするだけで、即座に高機能な16:9のHTMLスライドを動的に生成・閲覧・プレゼン・出力できる、完全ローカル完結型のWebスライド生成システムです。

旧バージョンの「ビルド時に原稿やJS/CSSを1枚のHTMLに固定する方式」から進化し、**ブラウザ上（クライアントサイド）での動的コンパイル、双方向マルチウィンドウ同期、および高度な前処理を伴うPPTXエクスポート**をサポートしたモダンなアーキテクチャに刷新されました。

---

## 🚀 主な機能

### 1. ゼロ構成のドラッグ＆ドロップ・インポート

- Markdownファイル（`.md`）や画像アセット（`png`, `jpeg`, `gif`, `svg`, `webp`）、独自の `css` ファイルをまとめてブラウザにドロップするだけで、インメモリに展開されて即座にスライド化されます。
- **ローカル環境（`file://`）対応**: ブラウザのセキュリティ制限（日本語パス制限など）によるフォルダ解析エラーを検知すると、自動的にファイルをフラットに吸い上げるフォールバックモードへ切り替わります。

### 2. 高度な拡張Markdownエンジン (`SlideEngine2`)

- **メタデータ・変数置換**: 原稿先頭のメタデータからタイトルや変数を抽出し、スライド内の `{% page-number %}` や `{% page-total %}` などのシステム変数やカスタム変数を動的に再帰置換します。置換後にインラインパースを行うため、変数内のMarkdown構文も正しくレンダリングされます。
- **スコープ付きスタイル**: 各スライド固有、または共通のスタイルは、CSSの汚染を防ぐため自動的に `@scope` でカプセル化されて注入されます。
- **数式・構文ハイライト**: `markdown-it-mathjax3` による高精度な数式表示（MathJax）と、`highlight.js` によるコードブロックのシンタックスハイライトに標準対応。
- **コロンブロック拡張 (`ColonBlockPlugin`)**: `:: [tag] [attrs] :: content`（単一形式）および `::: [tag] [attrs] \n inner \n :::`（複数行形式）に対応し、クラス名やスタイル属性を柔軟に付与したHTML構造をMarkdownから直接生成できます。

### 3. スマートなマルチウィンドウ連携（プレゼンターモード）

- 「プレゼン」ボタンを押すと、Blob URL（`file://` 環境でのセキュリティーエラーを回避する仕組み）を用いて、発表者専用の別ウィンドウ（プレゼンター画面）が立ち上がります。
- メインビューアーのスクロール位置と、プレゼンター画面でのキー操作（矢印キー、スペース、Enterなどによるページ送り・戻し）が `postMessage` を介して双方向にリアルタイム同期します。

### 4. 職人技的な前処理を伴う高精度PPTXエクスポート

- `dom-to-pptx` をベースに、PowerPoint書き出し時の表示崩れを防ぐための強力な自動前処理（DOMの最適化）を行います。
- **コードブロックの画像化**: ハイライトやインデント、フォント、高さを完全に再現するため、`html2canvas` を用いて高解像度（`scale: 2`）のPNG画像へ自動置換します。
- **MathJax (SVG) の px置換・クレンジング**: アクセシビリティ用の非表示MathML（`math` タグなど）を完全に除去してゴミテキストの混入を防ぐとともに、SVGのサイズ単位（`ex`など）をバウンディングボックスから計算した絶対 `px` 単位に書き換えることで、正確なベクター（SVG as Vector）としてPPTXへ書き出します。

### 5. 閲覧ビューアー・印刷（PDF）最適化

- **レスポンシブ・ズーム**: 基準サイズ（1280 × 720）に対して、ブラウザサイズに合わせた自動フィット（`Fit Height` / `Fit Width`）、等倍表示（`Original`）、ボタンによる10%刻みのカスタムズームに対応。
- **環境適応型アセットプロバイダー**: 実行環境を自動識別し、開発時は `httpProvider`（Fetch API経由）、ビルド後は `portableProvider`（単一HTML内に文字列として内包された `EmbeddedAssets` 経由）に透過的に切り替わります。
- **印刷・PDF化**: `Ctrl + P` または「印刷」ボタンにより、ツールバーや不要なコンポーネントを隠した状態で、スライドごとの綺麗なPDFとして出力・保存が可能です。

## 🛠️ 開発とビルド

本プロジェクトはランタイムおよびビルドツールとして [Bun](https://bun.sh/) を使用しています。

### 開発サーバーの起動

```bash
bun run dev.js

```

- `http://localhost:3000` でローカルサーバーが起動します。
- スクリプトやテーマCSS、テスト用のMarkdown原稿が変更されると、WebSocketを通じてブラウザが自動的にリロード（Live Reload）されます。

### スタンドアロンHTMLのビルド

```bash
bun run build.js

```

- 各種TypeScriptのコンパイルおよびCSS/HTMLテンプレートがシリアライズされ、**生のテキスト（文字列）データ**として `globalThis.EmbeddedAssets` オブジェクトに一括結合されます。
- 出力先: `./dist/index.html`
- 生成された1枚のHTMLファイル単体で、完全オフラインかつサーバーレス環境でも全機能が動作します。

## 🛠️ ユーザー操作・システム仕様

### 1. スライドの生成方法

1. システムを立ち上げたブラウザの画面（ドロップゾーン）に、作成したMarkdownファイル（`.md`）をドラッグ＆ドロップします。
2. Markdown内で相対パス指定（例: `./images/photo.png`）している画像アセットや、追加で適用したいカスタムCSS（`.css`）がある場合は、**Markdownファイルと一緒にまとめて複数選択してドロップ**、または**フォルダごとドロップ**してください。
3. パースが成功すると自動的にドロップゾーンが非表示になり、1ページ目のスライドが表示されます。（※ファイル名やパスが重複している場合や、Markdownが含まれない場合はアラートで通知されます）

### 2. ビューアーUIの操作

- **表示モード**: 画面上部のツールバーで切り替え可能です。
- `Original`: スライドを等倍（1280x720）で固定表示します。
- `Fit Width`: ウィンドウの横幅に合わせてスライドを拡大縮小します。
- `Fit Height` (デフォルト): ウィンドウの縦横比を考慮し、縦幅に綺麗に収まるように自動調整します。
- `Zoom In (+)` / `Zoom Out (-)`: 10%ずつ任意のサイズに調整します。

- **ページナビゲーション**:
- 縦スクロールするだけで、現在見ているページ位置をシステムが自動検知し、ツールバーのページ入力欄がリアルタイムに更新されます。
- ページ入力欄に直接数値を入力して `Enter` を押すことで、そのページへスムーズスクロールでジャンプできます。

### 3. プレゼンテーションの進行

1. ツールバーの「プレゼン」ボタンを押すと、プレゼンター用の独立したポップアップウィンドウが開きます。
2. このウィンドウをプロジェクターや外部ディスプレイ（全画面表示 F11 を推奨）に配置し、手元のメインビューアーで進捗を確認しながらプレゼンを行えます。
3. プレゼンター画面、またはビューアー画面のどちらかでページ操作（`ArrowRight` / `Space` / `PageDown` / `Enter` で進む、`ArrowLeft` / `Backspace` / `PageUp` で戻る）を行うと、もう一方の画面も完全に同期してスクロールします。

### 4. PPTX / PDF エクスポート

- **PPTXに出力**: 「pptxに出力」ボタンを押すと、バックグラウンドで数式のベクター最適化やコードブロックの画像高解像度化が実行され、少し待つと `{Markdownタイトル}.pptx` として自動ダウンロードされます。
- **PDF保存 / 印刷**: 「印刷」ボタンを押すか `Ctrl + P` を入力すると、印刷プレビューが開きます。送信先を「PDFに保存」に指定することで、16:9のレイアウトを維持した綺麗なPDFスライド書類が生成されます。

## 📝 Markdown 記述の仕様

### ページの区切り

スライドのページ区切りは、独立した行の `---`（水平線）で記述します。

```markdown
# 1ページ目のタイトル

ここに内容が入ります。

---

# 2ページ目のタイトル

ページが切り替わりました。
```

### コロンブロックによるHTML拡張 (`ColonBlockPlugin`)

特定の要素を特定のHTMLタグで囲ったり、クラスやインラインスタイルを付与したい場合に便利です。

#### ① 単一ライン形式 (`::`)

`::[タグ名] [属性] :: [コンテンツ]` の形式で記述します。タグ名を省略した場合は `div` になります。空白を含む属性値はダブルクォーテーションで囲みます。

```markdown
これは :: span class="highlight" style="color: red;" :: 強調したいテキスト :: です。
```

#### ② 複数行コンテナ形式 (`:::`)

ブロック形式のコンテナを生成します。終了時も同じ数のコロン（`::: `）で閉じます(省略が可能)。内部にはさらに別のMarkdownや数式をネスト可能です。

```markdown
:::section id="features" class="grid-layout"

### 特徴セクション

- リストアイテム1
- リストアイテム2

:::
```

### システム変数

スライド内で以下の記述を行うと、実行時に動的な値へ置換されます。

- `{% page-number %}` : 現在のページ番号 (1始まり)
- `{% page-total %}` : スライドの総ページ数
- メタデータ領域で定義した任意のカスタム変数

本システムは以下のライブラリ群をブラウザ上で有機的に結合して動作しています。

- **コアパース・拡張**: [markdown-it](https://github.com/markdown-it/markdown-it)

- **DOMキャプチャ**: [html2canvas](https://github.com/niklasvh/html2canvas)
- **ランタイム・ツールチェーン**: [Bun](https://bun.sh/)
