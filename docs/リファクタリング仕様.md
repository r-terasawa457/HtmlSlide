# 📑 【詳細仕様書】AssetProvider 抽象化アーキテクチャ (改訂版)

## 1. 目的と設計方針

- **関心の分離**: スライド表示やウィンドウ制御などの「コア・ランタイム・ロジック」から、環境依存（ファイルの調達方法）を完全に排除する。
- **データフローの適正化（事前Fetch方式）**: `SlideEngine2` の同期的なレンダリング処理との衝突を防ぐため、テーマCSSなどの非同期アセットは、描画を開始する前に呼び出し側（`viewer.ts`）で一括解決（事前Fetch）してエンジンに引き渡す。
- **単方向依存の徹底（循環参照の防止）**: サブ画面（`presenter.ts`, `pptxExport.ts`）は、親（`viewer.ts`）から `postMessage` 経由で完成したデータを受け取る受動的設計を維持する。サブ画面側から `AssetProvider` への依存（インポート）を一切禁止し、ビルド時の鶏と卵の罠を防止する。
- **メモリ管理の適正化（Blob URL キャッシュ）**: ポータブル版（スタンドアロン版）において、アセット解決のたびに Blob URL が累積するのを防ぐため、プロバイダー内部にインメモリのキャッシュ機構を導入する。
- **通信Origin制限の回避**: `file://` 環境における不透明なOrigin（Opaque Origin）の制約を回避するため、ウィンドウ間の `postMessage` 通信ではターゲットオリジンに必ず `"*"`（ワイルドカード）を使用する。

## 3. インターフェース定義 (`src/scripts/AssetProvider/types.ts`)

```typescript
/**
 * 各実行環境におけるシステムアセットの調達・解決を行うプロバイダーのインターフェース
 */
export interface IAssetProvider {
  /**
   * 指定された静的アセットパスを、現在の環境でブラウザが解釈可能なURLに解決する
   * @param path - 静的アセットの相対パス
   * @returns ブラウザで直接利用可能なURL文字列のPromise
   */
  resolveAssetUrl(path: string): Promise<string>;

  /**
   * 指定されたCSSファイルを、現在の環境に最適なHTMLタグ形式に解決する
   * @param path - スタイルシートの相対パス
   * @returns 展開可能なHTMLタグ文字列のPromise
   */
  resolveStyleTag(path: string): Promise<string>;

  /**
   * 指定されたテーマCSSの生テキスト内容を解決する
   * @param name - テーマ名、またはテーマファイルのパス
   * @returns CSSの生テキスト内容のPromise
   */
  resolveThemeCss(name: string): Promise<string>;
}
```

## 4. プロバイダー別の詳細仕様

### 4.1. `httpProvider.ts` (高機能HTTP版)

```typescript
import { IAssetProvider } from "./types";

export const httpProvider: IAssetProvider = {
  async resolveAssetUrl(path: string): Promise<string> {
    return "/" + path;
  },

  async resolveStyleTag(path: string): Promise<string> {
    return `<link rel="stylesheet" href="/${path}" />`;
  },

  async resolveThemeCss(name: string): Promise<string> {
    const response = await fetch(`/themes/${name}`);
    return response.text();
  },
};
```

### 4.2. `portableProvider.ts` (ポータブル・スタンドアロン版)

```typescript
import { IAssetProvider } from "./types";

declare const EmbeddedAssets: Record<string, string>;

const blobUrlCache: Record<string, string> = {};

export const portableProvider: IAssetProvider = {
  async resolveAssetUrl(path: string): Promise<string> {
    if (blobUrlCache[path]) {
      return blobUrlCache[path];
    }
    const content = EmbeddedAssets[path];
    const blob = new Blob([content], { type: "text/html" });
    const url = URL.createObjectURL(blob);
    blobUrlCache[path] = url;
    return url;
  },

  async resolveStyleTag(path: string): Promise<string> {
    const content = EmbeddedAssets[path];
    return `<style>${content}</style>`;
  },

  async resolveThemeCss(name: string): Promise<string> {
    return EmbeddedAssets[`themes/${name}`] || "";
  },
};
```

## 5. ランタイム連携仕様（事前Fetchと通信）

### 5.1. `viewer.ts` での事前Fetchと描画パイプライン

`SlideEngine2.run` は完全に同期処理として保つため、`viewer.ts` 側で非同期アセットを事前に解決してから流し込みます。

```typescript
// viewer.ts 内の描画ロジックのイメージ
async function renderSlides(
  markdownText: string,
  themeName: string,
): Promise<void> {
  const themeCss = await AssetProvider.resolveThemeCss(themeName);

  const result = SlidesEngine.run(markdownText, assetsMap, themeCss);
  setupViewer(result.html);
}
```

### 5.3. ウィンドウ間通信のセキュリティ

`file://` および `blob:` 間の Origin 制約を回避するため、通信時はターゲットオリジンを制限しません。

```typescript
presenterWindow.postMessage(
  { type: "sync_page", page: globalCurrentPage },
  "*",
);
```

# 🛠️ 【安全な移行・実装計画書】（改訂版）

```
【ステップ1】基盤・型定義とキャッシュ機構の作成
      │
【ステップ2】開発環境(dev.js)のルーティング整備 ➔ [検証1: 各種HTMLが404にならず配信できること]
      │
【ステップ3】ランタイム(viewer.ts)の事前Fetchリファクタ ➔ [検証2: 同期エンジンへのテーマCSS流し込み成否]
      │
【ステップ4】ビルドパイプライン(build.js)のエイリアス・インラインパッキング改修
      │
【ステップ5】ポータブル版の最終メモリ・ Origin検証 ➔ [検証3: メモリリークなし、ポップアップ同期正常]

```

### 【ステップ1〜2】開発環境の整備

- `portableProvider.ts` に仕様書通りのインメモリキャッシュ（`blobUrlCache`）を実装。
- `dev.js` の `Bun.serve` の `fetch` ハンドラを改修。`/dist/presenter.html` や `/dist/pptx_export.html` へのリクエストが、コピータスクで配置されたディスク上のファイルを正確に返す（404にならない）ルーティングになっていることを入念に確認・テスト。

### 【ステップ3】ランタイムの事前Fetchリファクタリング

- `SlideEngine2.ts` から `__BUILTIN_THEMES_PLACEHOLDER__` のパースや非同期処理の関心を完全に除去。
- `viewer.ts` 側を `async/await` ベースにリファクタリングし、スライドのドラッグ＆ドロップ時、またはテーマ切り替え時に **`AssetProvider.resolveThemeCss()` を `await` で呼んでから `SlidesEngine.run()` へ同期引数として渡す** 構造に集約。
- この段階で `localhost:3000` 上で全機能が従来通り動くことを検証。

### 【ステップ4〜5】パッキング改修とポータブル版検証

- `build.js` 内で `presenter.ts` と `pptxExport.ts` を単独先行ビルド。この2つのソースコードから `AssetProvider` への参照がない（依存が単方向である）ため、循環参照を起こさずにビルド可能。
- 生成されたシングル `index.html` を `file://` で開き、プレゼンター画面へのポップアップ・ページ同期が `*` 指定によってセキュリティブロックされずに動くこと、およびボタンを連打しても Blob URL の生成がキャッシュによって1度きりに抑えられていることを確認。
