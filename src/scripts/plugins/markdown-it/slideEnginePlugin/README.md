# slideEnginePlugin

SlideEngine2 の内部で markdown-it のパーサー・レンダラー拡張として動作し、Markdown からスライド用構造 (HTML/CSS) を生成する統合プラグイン。

## 仕様・動作概要

Markdown トークンストリームに対し、以下の3つのフェーズを順次適用してスライド用の構造とデータを生成・抽出します。

1. **メタデータパース (`MetaParser`)**:
   最初の区切り線（`---` など）より前の領域をメタセクションとして走査し、YAML メタ、`<style>`、タイトル、共通のヘッダー・フッター等を抽出して `SlideEnv` コンテキストに格納します。抽出後のメタトークンはストリームから完全に除去されます。
2. **スライド構造化・トランスフォーム (`StructureTransformer`)**:
   `---`（水平罫線）トークンをページ境界としてトークンストリームを分割し、各ページを `<section class="page" id="slide-X" data-page="X">` と `<div class="content">` でラッピングします。また、各ページ固有のヘッダー/フッターの検出および、共通パーツの自動補完・インジェクションを行います。
3. **変数置換 (`VariableReplacer`)**:
   全トークンのインラインテキスト、属性値、および子トークン内の変数プレースホルダー（`{% variable %}` 形式）を、システム予約変数（スライド番号等）やユーザー定義変数へと動的に再帰置換します。

---

## 各モジュールの実装詳細

### 1. `MetaParser.ts`

- **役割**: メタセクションの抽出と、描画用トークンストリームからの分離。
- **詳細**:
  - `colon_block_open` かつ `tag === "title"` のトークンからスライドタイトルを抽出します。
  - `<style>` ブロック内のCSSから、外側のスタイルタグを剥ぎ取って `StyleProcessor` を通し、`env.themeStyles` へ格納します。
  - `yaml` 指定されたコードブロックから YAML データを読み込み、`env.variables` にマージします。

### 2. `StructureTransformer.ts`

- **役割**: フラットなトークンストリームを、ページごとの `section.page` 構造へと再構築。
- **詳細**:
  - `hr` トークンを区切りとしてページ分割を実行します。
  - 各ページのトップレベル要素から、固有のヘッダー（`<header>` 等）やフッター（`<footer>` 等）を抽出し、コンテンツ領域（`div.content`）の外側に適切に配置します。
  - 固有の定義が無いページには、`MetaParser` で抽出した共通ヘッダー・フッターをディープコピーして自動挿入します。

### 3. `VariableReplacer.ts`

- **役割**: スライド内のカスタム変数プレースホルダーの全置換。
- **詳細**:
  - `{% variable.path %}` 形式のプレースホルダーを再帰的に解決します。
  - `page-number` や `page-total` などの動的システム変数に対応しています。
  - タグの属性（例: `class="{% theme %}"`）やネストされたオブジェクト値（スペース区切り文字列化）にも完全対応しています。

### 4. `StyleProcessor.ts`

- **役割**: スタイルシート内の `@import` ルールを解決し、インラインCSSへ再帰展開。
- **詳細**:
  - `assets` または `builtinThemes` 定義に基づき、ローカルな `@import` パスを実体CSSテキストへと置換します。
  - **ループ・重複防止**: グローバルに共有されるインポート履歴（`visited: Set<string>`）を使用します。同一ファイルが複数回読み込まれる場合や、循環参照（相互インポート）を検出した場合は、2回目以降の展開をスキップ（空文字列化）して無限再帰とスタイル重複を防止します。
  - 外部のリモートURL (`http://`, `https://`) やCSSコメント内の `@import` は無視してそのまま維持します。
